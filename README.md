# 社区慢病随访复核看板

本地社区卫生服务中心慢病随访数据质量核查与异常复核管理系统。

## 功能特性

- 📥 **三类数据导入**: 居民名册、预约计划、随访记录 CSV 文件导入
- 🔍 **五类异常自动识别**:
  - 逾期未访（预约已过但无随访）
  - 未预约到访（有随访但无对应预约）
  - 重复随访（同一居民同一天多条随访）
  - 指标越界（血压/血糖超出正常范围）
  - 居民不在名册（记录的居民编号不存在）
- 🎛️ **多维筛选**: 按站点、护士、异常类型、复核状态筛选
- ✅ **异常复核**: 标记待处理、已确认、忽略、需上门四种状态，支持备注和处理人
- 💾 **本地持久化**: 所有数据（含复核状态、备注）存储在 localStorage，刷新重启不丢失
- 📤 **数据导出**: JSON/CSV 格式导出当前筛选结果

## 快速开始

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build
```

访问 `http://localhost:5173` 查看应用。

## 操作流程

1. 依次导入 `src/sample-data/` 目录下的三个 CSV 文件（居民名册 → 预约计划 → 随访记录）
2. 系统自动识别异常并展示在看板
3. 使用左侧筛选栏按站点、护士、异常类型等筛选
4. 点击异常行左侧展开箭头，查看详情、选择复核状态、填写处理人和备注
5. 点击右上角"导出"按钮，选择 CSV 或 JSON 格式导出当前筛选结果

## CSV 字段要求

### 居民名册 (residents.csv)
| 字段 | 必填 | 说明 |
|------|------|------|
| 居民编号 | ✅ | 唯一标识 |
| 姓名 | ✅ | 居民姓名 |
| 性别 | - | 男/女 |
| 所属站点 | - | 如：中心站、东站、西站 |
| 责任护士 | - | 护士姓名 |
| 慢病类型 | - | 如：高血压、糖尿病 |

### 预约计划 (appointments.csv)
| 字段 | 必填 | 说明 |
|------|------|------|
| 居民编号 | ✅ | 必须在居民名册中存在 |
| 预约日期 | ✅ | 格式：YYYY-MM-DD |
| 执行护士 | - | 护士姓名 |
| 站点 | - | 所属站点 |
| 状态 | - | 已预约/已完成/已取消 |

### 随访记录 (followups.csv)
| 字段 | 必填 | 说明 |
|------|------|------|
| 居民编号 | ✅ | 必须在居民名册中存在 |
| 随访日期 | ✅ | 格式：YYYY-MM-DD |
| 随访护士 | - | 护士姓名 |
| 站点 | - | 所属站点 |
| 收缩压 | - | 数字，单位 mmHg，正常 ≤140 |
| 舒张压 | - | 数字，单位 mmHg，正常 ≤90 |
| 血糖 | - | 数字，单位 mmol/L，正常 ≤7.0 |
| 到访方式 | - | 如：门诊、上门 |

## 三类异常复现方法

### 1. 坏日期格式 — 导入不通过，保留旧数据

**测试文件**: `src/sample-data/test/appointments_bad_date.csv`

**文件内容**:
```csv
居民编号,预约日期,执行护士,站点,状态
R001,2026-05-20,李护士,中心站,已预约
R002,not-a-date,李护士,中心站,已预约
R003,2026-13-45,王护士,东站,已预约
```

**复现步骤**:
1. 先导入正常的预约计划 `appointments.csv`（确保有旧数据）
2. 再导入 `test/appointments_bad_date.csv`
3. 观察：卡片显示"导入失败"，提示日期格式错误
4. 验证：顶部数据统计中"预约"数量不变，旧数据未被清空

**校验逻辑**:
- 支持格式：`YYYY-MM-DD`、`YYYY/MM/DD`、`YYYY年M月D日`
- 无效格式或不存在的日期（如 13月45日）会导致整批数据拒绝导入

---

### 2. 非数字指标 — 导入不通过，保留旧数据

**测试文件**: `src/sample-data/test/followups_bad_metrics.csv`

**文件内容**:
```csv
居民编号,随访日期,随访护士,站点,收缩压,舒张压,血糖,到访方式
R001,2026-05-22,李护士,中心站,138,85,6.2,门诊
R002,2026-05-20,李护士,中心站,abc,95,7.8,门诊
R003,2026-06-02,王护士,东站,142,xyz,5.9,门诊
R004,2026-05-28,王护士,东站,128,80,不是数字,上门
```

**复现步骤**:
1. 先导入正常的随访记录 `followups.csv`
2. 再导入 `test/followups_bad_metrics.csv`
3. 观察：卡片显示"导入失败"，提示具体字段（收缩压/舒张压/血糖）必须为数字
4. 验证：顶部"随访"数量保持不变，异常列表数据未变化

**校验逻辑**:
- 数值字段（收缩压、舒张压、血糖）若填写则必须为合法数字
- 空值允许（表示未测量），但填写了非数字内容则校验失败

---

### 3. 缺少居民编号 — 导入不通过，保留旧数据

**测试文件**: `src/sample-data/test/residents_missing_id.csv`

**文件内容**:
```csv
居民编号,姓名,性别,所属站点,责任护士,慢病类型
R001,张建国,男,中心站,李护士,高血压
,王秀兰,女,中心站,李护士,糖尿病
R003,刘德明,男,东站,王护士,高血压
```

**复现步骤**:
1. 先导入正常的居民名册 `residents.csv`
2. 再导入 `test/residents_missing_id.csv`
3. 观察：卡片显示"导入失败"，提示"缺少必填字段: 居民编号"并显示行号
4. 验证：居民列表仍为 10 人，王秀兰（无编号）未被导入，旧数据完好

**校验逻辑**:
- 居民编号是三类数据的核心关联字段，缺失将导致无法匹配
- 任何一条记录缺少必填字段，整批数据均拒绝导入

## 其他验收要点验证

### 重复导入同一文件不重复计数

1. 导入 `residents.csv`，记录导入成功提示和居民数量
2. 再次导入同一个 `residents.csv`
3. 观察：卡片显示"重复文件"提示，居民数量不变

### 未登记居民单独列出

使用正常样例数据导入后：
- 在"未登记居民记录"卡片中可看到 R888（来自随访记录）和 R999（来自预约记录）
- 在异常列表的"居民不在名册"类型中也可筛选到对应记录

### 重启后状态恢复

1. 导入样例数据，对某条异常标记状态（如改为"已确认"）、填写备注和处理人
2. 刷新浏览器页面（F5）
3. 验证：所有数据、异常列表、复核状态、备注、处理人、更新时间均与刷新前一致

### 导出内容与筛选结果一致

1. 设置某些筛选条件（如选择站点"中心站"、异常类型"指标越界"）
2. 记录异常列表中显示的条目数量
3. 点击导出 → CSV 或 JSON
4. 打开导出文件，验证记录数量与页面显示一致，且均符合筛选条件

## 技术栈

- **框架**: React 18 + TypeScript
- **构建**: Vite 6
- **样式**: Tailwind CSS 3
- **状态管理**: Zustand 5 (含 localStorage 持久化)
- **图标**: Lucide React
- **纯前端**: 无需后端服务器，数据全部本地存储
