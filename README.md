# 社区慢病随访复核看板

本地社区卫生服务中心慢病随访数据质量核查与异常复核管理系统。

## 功能特性

- 📥 **三类数据导入**: 居民名册、预约计划、随访记录 CSV 文件导入
- 🔍 **五类异常自动识别**:
  - 逾期未访（预约已过但无随访）
  - 未预约到访（有随访但无对应预约）
  - 重复随访（同一居民同一天多条随访）
  - 指标越界（血压/血糖超出正常范围）
  - 居民不在名册（记录的居民编号不存在）
- 🎛️ **多维筛选**: 按站点、护士、异常类型、复核状态筛选
- ✅ **异常复核**: 标记待处理、已确认、忽略、需上门四种状态，支持备注和处理人
- 🛡️ **质控规则配置**: 可配置逾期阈值、血压血糖上下限、未预约匹配窗口等规则，支持版本管理
- ⏮️ **规则回放**: 规则版本历史管理，预览差异后切换版本，保护人工复核结果
- 📒 **操作日志**: 记录导入、规则变更、批量重算、导出、复核状态修改，支持筛选导出
- 💾 **本地持久化**: 所有数据（含复核状态、规则、日志）存储在 localStorage，刷新重启不丢失
- 📤 **数据导出**: JSON/CSV 格式导出当前筛选结果

## 功能模块

### 三大模块导航

系统通过顶部 Tab 切换三大模块：
- **异常看板**: 数据导入、异常列表展示、筛选与复核
- **质控规则**: 规则配置、版本管理、批量重算
- **操作日志**: 操作记录查询、筛选与导出

## 质控规则配置

### 可配置规则项

| 规则项 | 说明 | 默认值 |
|--------|------|--------|
| 规则启用开关 | 关闭后仅识别"重复随访"和"居民不在名册" | 启用 |
| 逾期未访阈值 | 超过多少天未随访算逾期 | 0天 |
| 未预约匹配窗口 | 随访前后多少天内有预约视为匹配 | 3天 |
| 收缩压上限/下限 | 收缩压正常范围 | 90-140 mmHg |
| 舒张压上限/下限 | 舒张压正常范围 | 60-90 mmHg |
| 血糖上限/下限 | 血糖正常范围 | 3.9-7.0 mmol/L |
| 需要上门的状态映射 | 哪些异常类型在首次识别时，复核状态自动标记为"需上门"而非默认的"待处理" | 逾期未访、指标越界 |

### 操作流程

1. **修改规则**: 在"质控规则" Tab 调整各项阈值
2. **预览差异**: 点击"预览差异"查看规则变更对现有异常的影响
3. **保存版本**: 填写版本名称后点击"保存并应用"，自动生成新版本
4. **恢复默认**: 点击"恢复默认"一键回到出厂规则

5. 验证：在"未登记居民记录"卡片中可看到 R888 和 R999

### 二、质控规则配置与生效

**验收目标**: 修改规则后异常识别立即按新规则计算

**复现步骤**:
1. 导入样例数据后，切换到"质控规则" Tab
2. 记录当前"指标越界"类异常数量
3. 将"收缩压上限"从 140 改为 180（放宽标准）
4. 点击"预览差异"
5. 观察：弹窗显示"新增异常 0"、"移除异常 N"（N 为收缩压在 141-179 之间的记录数）
6. 点击"确认应用"
7. 回到"异常看板"，验证：指标越界类异常数量相应减少
8. 将"血糖上限"从 7.0 改为 11.0（放宽标准）
9. 不预览，直接"保存并应用"，填写版本名"放宽血糖标准"
10. 验证：异常数量再次变化，版本历史中可看到新版本

### 三、规则版本管理与回放

**验收目标**: 版本可追溯、可切换，切换时有差异预览

**复现步骤**:
1. 确保已创建至少一个自定义规则版本
2. 在"质控规则" Tab 点击右上角"版本历史"
3. 观察：列出所有版本，包括"默认规则 v1.0.0-default"和自定义版本
4. 点击某个非当前版本的"应用"按钮
5. 观察：弹出差异预览窗口，显示切换后新增/移除/受保护的异常数量
6. 点击"确认应用"
7. 验证：异常列表按选中版本规则重新计算，顶部显示当前版本名称
8. 再次打开版本历史，验证：被删除按钮仅对自定义版本可用，默认版本不可删除

---

### 四、人工复核结果保护（冲突处理）

**验收目标**: 已确认/忽略的异常不会被规则变更或重算静默覆盖

**复现步骤**:
1. 导入样例数据，在异常列表中选择 2-3 条异常
2. 将其中一条状态改为"已确认"，另一条改为"忽略"，第三条保留"待处理"
3. 切换到"质控规则" Tab，大幅度修改规则（如将血压上下限设为极端值）
4. 点击"保存并应用"
5. 回到"异常看板"，找到刚才操作的三条记录
6. 验证：
   - 状态为"已确认"和"忽略"的记录，状态和备注完好无损
   - 即使按新规则该记录不应被识别为异常，它依然保留在列表中（受保护）
   - 状态为"待处理"的记录，按新规则正常更新
7. 点击顶部"重算"按钮，选择"保护状态并重算"
8. 再次验证：已确认/忽略的记录仍然存在且状态未变

## 操作流程（完整闭环）

1. 设置操作者姓名（操作日志右上角输入框）
2. 依次导入 `src/sample-data/` 目录下的三个 CSV 文件
3. 切换到"质控规则" Tab，根据需要调整规则阈值并保存版本

## CSV 字段要求

### 居民名册 (residents.csv)
| 字段 | 必填 | 说明 |
|------|------|------|
| 居民编号 | ✅ | 唯一标识 |
| 姓名 | ✅ | 居民姓名 |
| 性别 | - | 男/女 |
| 所属站点 | - | 如：中心站、东站、西站 |
| 责任护士 | - | 护士姓名 |
| 慢病类型 | - | 如：高血压、糖尿病 |

### 预约计划 (appointments.csv)
| 字段 | 必填 | 说明 |
|------|------|------|
| 居民编号 | ✅ | 必须在居民名册中存在 |
| 预约日期 | ✅ | 格式：YYYY-MM-DD |
| 执行护士 | - | 护士姓名 |
| 站点 | - | 所属站点 |
| 状态 | - | 已预约/已完成/已取消 |

### 随访记录 (followups.csv)
| 字段 | 必填 | 说明 |
|------|------|------|
| 居民编号 | ✅ | 必须在居民名册中存在 |
| 随访日期 | ✅ | 格式：YYYY-MM-DD |
| 随访护士 | - | 护士姓名 |
| 站点 | - | 所属站点 |
| 收缩压 | - | 数字，单位 mmHg，正常范围由质控规则决定 |
| 舒张压 | - | 数字，单位 mmHg，正常范围由质控规则决定 |

## 三类异常复现方法（原有功能）

### 1. 坏日期格式 — 导入不通过，保留旧数据

**测试文件**: `src/sample-data/test/appointments_bad_date.csv`

**复现步骤**:
1. 先导入正常的预约计划 `appointments.csv`（确保有旧数据）
2. 再导入 `test/appointments_bad_date.csv`
3. 观察：卡片显示"导入失败"，提示日期格式错误
4. 验证：顶部数据统计中"预约"数量不变，旧数据未被清空

### 2. 非数字指标 — 导入不通过，保留旧数据

**测试文件**: `src/sample-data/test/followups_bad_metrics.csv`

**复现步骤**:
1. 先导入正常的随访记录 `followups.csv`
2. 再导入 `test/followups_bad_metrics.csv`
3. 观察：卡片显示"导入失败"，提示具体字段必须为数字
4. 验证：顶部"随访"数量保持不变，异常列表数据未变化

### 3. 缺少居民编号 — 导入不通过，保留旧数据

**测试文件**: `src/sample-data/test/residents_missing_id.csv`

**复现步骤**:
1. 先导入正常的居民名册 `residents.csv`
2. 再导入 `test/residents_missing_id.csv`
3. 观察：卡片显示"导入失败"，提示缺少必填字段并显示行号
4. 验证：居民列表仍为 10 人，旧数据完好

## 其他验收要点

### 重复导入同一文件不重复计数
1. 导入 `residents.csv`，记录导入成功提示和居民数量
2. 再次导入同一个 `residents.csv`
3. 观察：卡片显示"重复文件"提示，居民数量不变

### 导出内容与筛选结果一致
1. 设置某些筛选条件（如选择站点"中心站"、异常类型"指标越界"）
2. 记录异常列表中显示的条目数量
3. 点击导出 → CSV 或 JSON
4. 打开导出文件，验证记录数量与页面显示一致，且均符合筛选条件
